# 기여 가이드라인

## 코드 리뷰 프로세스

### 1. 브랜치 전략

- `main`: 안정적인 릴리즈 브랜치
- `develop`: 개발 브랜치
- `feature/*`: 새로운 기능 개발
- `bugfix/*`: 버그 수정
- `hotfix/*`: 긴급 수정

### 2. Pull Request 규칙

1. 기능별로 브랜치를 생성합니다
2. 작업 완료 후 PR을 생성합니다
3. 코드 리뷰를 받습니다
4. 승인 후 main에 병합합니다

### 3. 커밋 메시지 규칙

```
<타입>: <제목>

<본문>
```

**타입 종류**:
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `docs`: 문서 수정
- `style`: 코드 포맷팅
- `refactor`: 코드 리팩토링
- `test`: 테스트 코드
- `chore`: 빌드, 설정 변경

### 4. 코드 스타일

- Unity C# 코딩 컨벤션을 따릅니다
- 변수명은 camelCase
- 클래스명은 PascalCase
- 주석은 한글로 작성 가능
